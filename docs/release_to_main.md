# Release roadmap: develop → main

This repository uses `develop` as the integration branch. When you want to publish a stable snapshot to `main`, use this checklist.

## 1. Pre-flight

- [ ] Ensure `develop` is green locally:
  - `uv run ruff check src/ tests/`
  - `uv run ruff format --check src/ tests/`
  - `uv run pytest -q`
- [ ] Ensure documentation is consistent:
  - README usage examples still work
  - `LICENSE` matches README badge and `pyproject.toml` metadata

## 2. Update GitHub “About” (repo settings)

Set description, homepage, and topics (example):

```bash
PAGER=cat GH_PAGER=cat gh repo edit R0SEWT/potato-achis \
  --description "Multi-source domain adaptation (MDFAN) for Andean potato disease classification" \
  --homepage "https://github.com/R0SEWT/potato-achis#readme" \
  --add-topic pytorch \
  --add-topic computer-vision \
  --add-topic domain-adaptation \
  --add-topic transfer-learning \
  --add-topic image-classification \
  --add-topic plant-disease \
  --add-topic potato \
  --add-topic ood-detection \
  --add-topic onnx
```

## 3. PR: merge `develop` into `main`

```bash
PAGER=cat GH_PAGER=cat gh pr create \
  --base main \
  --head develop \
  --title "Release: develop → main" \
  --body $'Summary\n- Promote the current `develop` state to `main`.'

PAGER=cat GH_PAGER=cat gh pr merge --auto --merge
```

## 4. Tag + GitHub Release

- [ ] Decide the version bump (project is currently `0.1.0` in `pyproject.toml`).
- [ ] Create a release with autogenerated notes:

```bash
PAGER=cat GH_PAGER=cat gh release create v0.1.0 --generate-notes
```

## 5. Post-release

- [ ] Sanity check quickstart commands on a clean machine/CI.
- [ ] If you publish packages later, verify `pyproject.toml` classifiers/metadata are correct.
